To build up the final target, there is some problem I should now figure:

Underdamped -- vibration on the haptic-gripper. Using velocity rather than direct positions. The feedback of the system should be well evaluated. The unstable system may sometimes dead.

System break down if there are so much threads, how would I develop the system to make them works better:
1. Thread for major movement
2. Thread for Loading/Passing Pressure
3. Thread for GUI

The Arduino Board, as we all know, this guy is a single thread system, where we cannot do multiple task on this. The haptic device consist of a Receiving Module (for the servo) and move the servo, in the time that moving the servo (around 300ms), the transmission on the next block will be delayed which is the pressure sensor data. It would increase the error of the control system.

Considering the hardware problem, a possible solution is putting the pressure vs displacement formula on all of the devices, where the haptic device just need to response to the pressure change. A kind of offline method, will be discussed in details in report.
